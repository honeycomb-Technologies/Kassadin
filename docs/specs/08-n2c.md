# Spec 08: Node-to-Client Protocols

## Overview

Node-to-client (N2C) protocols run over a UNIX domain socket and serve local applications: wallets, CLI tools, DApps. They are trusted (no size limits, no timeouts) and use the same multiplexer as N2N.

---

## 1. Local Chain-Sync (Protocol Num 5)

Identical to N2N Chain-Sync (Spec 03, Section 3) except:
- Transfers **full blocks** instead of headers
- No per-state size limits
- No timeouts
- UNIX domain socket bearer

The client follows the node's chain by requesting blocks one at a time. The intersection-finding mechanism allows a client to resume from where it left off.

---

## 2. Local Tx-Submission (Protocol Num 6)

Push-based transaction submission from local clients.

### Zig Types
```zig
pub const LocalTxSubmissionMsg = union(enum) {
    submit_tx: []const u8,       // CBOR-encoded transaction
    accept_tx: void,
    reject_tx: TxValidationError,
    done: void,
};
```

### Rejection Reasons
Era-specific validation errors. Must encode the full error structure so the client can understand why the transaction was rejected:
- Insufficient funds
- Invalid inputs (not in UTxO)
- Fee too low
- Validity interval expired
- Script execution failure (with details)
- Missing witnesses
- etc.

## 3. Local State Query (Protocol Num 7)

Query the node's ledger state at a specific chain point.

### Query Types (Era-Dependent)

The query is a CBOR-encoded structure that identifies what to query. Key queries:

```zig
pub const Query = union(enum) {
    // System-level
    system_start: void,
    hard_fork_eras: void,
    current_era: void,

    // Shelley+ queries (wrapped in era tag)
    ledger_tip: void,
    epoch_no: void,
    non_myopic_member_rewards: []const Credential,
    current_protocol_params: void,
    proposed_protocol_params_updates: void,
    stake_distribution: void,
    utxo_by_address: []const Address,
    utxo_whole: void,
    debug_epoch_state: void,
    cbor_wrap_debug_epoch_state: void,
    filtered_delegations_and_reward_accounts: []const Credential,
    genesis_config: void,
    debug_new_epoch_state: void,
    debug_chain_dep_state: void,
    reward_provenance: void,
    utxo_by_tx_in: []const TxIn,
    stake_pools: void,
    stake_pool_params: []const KeyHash,
    reward_info_pools: void,
    pool_state: []const KeyHash,
    stake_snapshots: []const KeyHash,
    pool_distr: void,

    // Conway queries
    gov_state: void,
    drep_state: []const Credential,
    drep_stake_distribution: []const DRep,
    committee_state: void,
    constitution: void,
    account_state: void,
};
```

### Query Encoding
The query is wrapped in era tags to route to the correct ledger era:
```
query = [era_tag, [inner_query_tag, ?params]]

Example: current protocol params in Conway era
  [6,    // Conway era tag
   [0,   // Shelley-based era queries
    [1]  // GetCurrentPParams
   ]
  ]
```

### Result Encoding
Each query returns an era-specific result:
- `currentProtocolParams` → full PParams CBOR
- `utxoByAddress` → Map TxIn TxOut
- `stakeDistribution` → Map PoolId (Coin, Coin)
- etc.

## 4. Local Tx-Monitor (Protocol Num 9)

Introspect the mempool contents.

### Key Operations
1. **Acquire** — get a consistent snapshot of the mempool
2. **NextTx** — iterate through transactions in the snapshot
3. **HasTx** — check if a specific TxId is in the mempool
4. **GetSizes** — get mempool capacity and usage
5. **Release** — release the snapshot

### Usage Pattern
```
Acquire → Acquired(slotNo)
  → NextTx → ReplyNextTx(Just tx)
  → NextTx → ReplyNextTx(Just tx)
  → NextTx → ReplyNextTx(Nothing)    // end of mempool
  → HasTx(txid) → ReplyHasTx(true)
  → GetSizes → ReplyGetSizes(cap, size, count)
  → Release
```

## 5. UNIX Socket Setup

### Socket Path
Configured via `--socket-path` CLI flag or `CARDANO_NODE_SOCKET_PATH` environment variable.

### Connection Handling
```zig
pub fn startLocalServer(socket_path: []const u8) !void {
    const server = try std.posix.socket(std.posix.AF.UNIX, std.posix.SOCK.STREAM, 0);
    defer std.posix.close(server);

    var addr = std.posix.sockaddr.un{ .path = socket_path };
    try std.posix.bind(server, &addr);
    try std.posix.listen(server, 5);

    while (true) {
        const client = try std.posix.accept(server);
        // Spawn handler with N2C handshake + multiplexer
        try spawnClientHandler(client);
    }
}
```

### N2C Handshake
Same as N2N handshake but with N2C version numbers (16-21) and simpler version data (just network_magic).
